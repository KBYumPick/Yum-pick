# User Stories - 냠픽(Yumpick)

## Epic 1: 테이블 인증 (Table Authentication)

### US-01: 테이블 초기 설정
- **As a** 매장 관리자
- **I want to** 테이블 태블릿에 매장 식별자, 테이블 번호, 비밀번호를 설정
- **So that** 고객이 별도 로그인 없이 주문할 수 있다
- **Acceptance Criteria**:
  - [ ] 매장 식별자, 테이블 번호, 비밀번호 입력 폼 제공
  - [ ] 로그인 성공 시 정보를 로컬에 저장
  - [ ] 저장된 정보로 자동 로그인 수행

### US-02: 테이블 자동 로그인
- **As a** 고객
- **I want to** 태블릿을 켜면 자동으로 로그인되어
- **So that** 바로 메뉴를 보고 주문할 수 있다
- **Acceptance Criteria**:
  - [ ] 저장된 로그인 정보로 자동 인증
  - [ ] 인증 실패 시 초기 설정 화면으로 이동
  - [ ] 16시간 세션 유지

---

## Epic 2: 메뉴 조회 (Menu Browsing)

### US-03: 카테고리별 메뉴 조회
- **As a** 고객
- **I want to** 카테고리별로 메뉴를 탐색
- **So that** 원하는 음식을 쉽게 찾을 수 있다
- **Acceptance Criteria**:
  - [ ] 메뉴 화면이 기본 화면으로 표시
  - [ ] 카테고리 탭으로 메뉴 분류
  - [ ] 카테고리 간 빠른 이동 가능

### US-04: 메뉴 상세 정보 확인
- **As a** 고객
- **I want to** 메뉴의 이름, 가격, 설명, 이미지를 확인
- **So that** 주문 전에 충분한 정보를 얻을 수 있다
- **Acceptance Criteria**:
  - [ ] 카드 형태 레이아웃으로 메뉴 표시
  - [ ] 메뉴명, 가격, 설명, 이미지 표시
  - [ ] 터치 친화적 버튼 크기 (최소 44x44px)

---

## Epic 3: 장바구니 (Cart)

### US-05: 장바구니에 메뉴 추가
- **As a** 고객
- **I want to** 원하는 메뉴를 장바구니에 담을 수 있어
- **So that** 여러 메뉴를 한번에 주문할 수 있다
- **Acceptance Criteria**:
  - [ ] 메뉴 카드에서 추가 버튼으로 장바구니에 담기
  - [ ] 장바구니에 담긴 수량 표시
  - [ ] 총 금액 실시간 계산

### US-06: 장바구니 수정
- **As a** 고객
- **I want to** 장바구니의 수량을 조절하거나 메뉴를 삭제
- **So that** 주문 전에 내용을 수정할 수 있다
- **Acceptance Criteria**:
  - [ ] 수량 증가/감소 버튼
  - [ ] 메뉴 삭제 기능
  - [ ] 장바구니 비우기 기능
  - [ ] 페이지 새로고침 시에도 장바구니 유지 (localStorage)

---

## Epic 4: 주문 (Order)

### US-07: 주문 생성
- **As a** 고객
- **I want to** 장바구니의 메뉴를 주문으로 확정
- **So that** 매장에서 음식을 준비해준다
- **Acceptance Criteria**:
  - [ ] 주문 내역 최종 확인 화면
  - [ ] 주문 확정 버튼
  - [ ] 주문 성공 시: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 리다이렉트
  - [ ] 주문 실패 시: 에러 메시지 표시, 장바구니 유지
  - [ ] 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID

### US-08: 주문 내역 조회
- **As a** 고객
- **I want to** 현재 테이블의 주문 내역을 확인
- **So that** 주문 상태와 금액을 알 수 있다
- **Acceptance Criteria**:
  - [ ] 현재 세션 주문만 표시
  - [ ] 주문 시간 순 정렬
  - [ ] 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태
  - [ ] 상태: 대기중/준비중/완료
  - [ ] 수동 새로고침으로 상태 업데이트

---

## Epic 5: 관리자 인증 (Admin Auth)

### US-09: 관리자 로그인
- **As a** 매장 관리자
- **I want to** 매장 식별자, 사용자명, 비밀번호로 로그인
- **So that** 매장 관리 시스템에 접근할 수 있다
- **Acceptance Criteria**:
  - [ ] 매장 식별자 + 사용자명 + 비밀번호 입력 폼
  - [ ] JWT 토큰 기반 인증
  - [ ] 16시간 세션 유지
  - [ ] 브라우저 새로고침 시 세션 유지
  - [ ] 16시간 후 자동 로그아웃
  - [ ] 비밀번호 bcrypt 해싱
  - [ ] 로그인 시도 제한

---

## Epic 6: 실시간 주문 모니터링 (Order Monitoring)

### US-10: 실시간 주문 대시보드
- **As a** 매장 관리자
- **I want to** 들어오는 주문을 실시간으로 확인
- **So that** 주문을 놓치지 않고 빠르게 처리할 수 있다
- **Acceptance Criteria**:
  - [ ] SSE 기반 실시간 주문 업데이트 (2초 이내)
  - [ ] 테이블별 카드 형태 그리드 레이아웃
  - [ ] 각 테이블 카드에 총 주문액 표시
  - [ ] 최신 주문 미리보기
  - [ ] 신규 주문 시각적 강조 (색상/애니메이션)

### US-11: 주문 상태 변경
- **As a** 매장 관리자
- **I want to** 주문 상태를 대기중 → 준비중 → 완료로 변경
- **So that** 주문 처리 진행 상황을 관리할 수 있다
- **Acceptance Criteria**:
  - [ ] 주문 카드 클릭 시 상세 보기
  - [ ] 상태 변경 버튼 (대기중 → 준비중 → 완료)
  - [ ] 테이블별 필터링

### US-12: 주문 삭제
- **As a** 매장 관리자
- **I want to** 잘못된 주문을 삭제
- **So that** 정확한 주문 관리를 할 수 있다
- **Acceptance Criteria**:
  - [ ] 확인 팝업 후 삭제
  - [ ] 삭제 후 총 주문액 재계산

---

## Epic 7: 테이블 관리 (Table Management)

### US-13: 테이블 등록/수정
- **As a** 매장 관리자
- **I want to** 테이블을 등록하고 비밀번호를 설정
- **So that** 고객이 해당 테이블에서 주문할 수 있다
- **Acceptance Criteria**:
  - [ ] 테이블 번호, 비밀번호 입력
  - [ ] 테이블 목록 조회
  - [ ] 테이블 정보 수정/삭제

### US-14: 테이블 이용 완료 (세션 종료)
- **As a** 매장 관리자
- **I want to** 고객 퇴장 시 테이블 세션을 종료
- **So that** 다음 고객을 위해 테이블을 초기화할 수 있다
- **Acceptance Criteria**:
  - [ ] 확인 팝업 후 세션 종료
  - [ ] 주문 내역 과거 이력으로 이동
  - [ ] 주문 목록/총액 리셋

### US-15: 과거 주문 내역 조회
- **As a** 매장 관리자
- **I want to** 테이블별 과거 주문 내역을 확인
- **So that** 매출과 주문 이력을 파악할 수 있다
- **Acceptance Criteria**:
  - [ ] 테이블별 과거 주문 목록 (시간 역순)
  - [ ] 날짜 필터링

---

## Epic 8: 메뉴 관리 (Menu Management)

### US-16: 메뉴 등록/수정/삭제
- **As a** 매장 관리자
- **I want to** 메뉴를 등록, 수정, 삭제
- **So that** 매장 메뉴를 최신 상태로 유지할 수 있다
- **Acceptance Criteria**:
  - [ ] 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 URL 입력
  - [ ] 메뉴 수정: 기존 정보 수정
  - [ ] 메뉴 삭제: 확인 후 삭제
  - [ ] 필수 필드 검증, 가격 범위 검증

### US-17: 메뉴 노출 순서 조정
- **As a** 매장 관리자
- **I want to** 메뉴의 노출 순서를 변경
- **So that** 인기 메뉴나 추천 메뉴를 상단에 배치할 수 있다
- **Acceptance Criteria**:
  - [ ] 드래그 앤 드롭 또는 순서 변경 UI
  - [ ] 변경된 순서 즉시 반영
